# StudyQuest ドキュメント

ドキュメント管理ディレクトリ

## ドキュメント一覧

| ファイル | 役割 | 主な読者 |
|---|---|---|
| `db/db-design.md` | DBの正式なテーブル定義 | バックエンド担当 |
| `db/db-guide.md` | DB設計の読み方・画面との対応・データの流れ | 全員 |
| `api/api-design.md` | フロントとバックエンドで合意するAPI仕様 | フロント・バックエンド担当 |

## 読む順番

1. `db/db-guide.md` を読む
2. `db/db-design.md` で正式なテーブル定義を確認する
3. `api/api-design.md` で画面とAPIのつながりを確認する

## 更新ルール

- テーブルやカラムを変更したら、必ず `db/db-design.md` を更新する。
- 画面で必要なAPIが増えたら、必ず `api/api-design.md` を更新する。
- 初学者向けの説明や例は `db/db-guide.md` にまとめる。
- 同じ説明を複数ファイルに重複して書きすぎない。
